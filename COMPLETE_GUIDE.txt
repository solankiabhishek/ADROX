INSTALLATION & SETUP
====================

STEP 1: INSTALL XAMPP
- Download XAMPP from https://www.apachefriends.org/
- Install to default location
- Start Apache and MySQL

STEP 2: LOCATE FILES
- Place all files in: C:\xampp\htdocs\adrox\

STEP 3: RUN SETUP
- Open browser
- Go to: http://localhost/adrox/setup.php
- Wait for setup to complete (should see green checkmarks)

STEP 4: ACCESS WEBSITE
- Website: http://localhost/adrox/
- Admin Panel: http://localhost/adrox/admin/
- Status Page: http://localhost/adrox/status.php

ADMIN PANEL GUIDE
=================

ACCESS: http://localhost/adrox/admin/
LOGIN: username: admin | password: admin123

FEATURES:
1. Dashboard - Overview of all products
2. Add Product - Add new affiliate products
3. Edit Product - Modify existing products
4. Delete Product - Remove products
5. Featured Products - Mark products for homepage display
6. Categories - Manage product categories

ADDING A PRODUCT:
1. Click "Add New Product" button
2. Fill in:
   - Product Name (e.g., "Premium Mobile Cover")
   - Description (detailed product info)
   - Price (₹)
   - Category (Mobile Covers, Car Keys, etc.)
   - Affiliate Link (where to redirect on "Buy")
   - Image URL (optional)
   - Featured (checkbox - shows on homepage)
3. Click "Save Product"

AFFILIATE LINK:
- This is the URL users go to when clicking "Buy Now"
- Could be Amazon, Flipkart, AliExpress, eBay, etc.
- Use your affiliate tracking link for earnings
- Example: https://amazon.in/s?k=mobile+cover&ref=adrox

DATABASE STRUCTURE
==================

DATABASE NAME: adrox_ecommerce

TABLES:
1. products
   - id: Product ID
   - name: Product name
   - description: Detailed description
   - price: Price (₹)
   - category: Product category
   - image_url: Image URL
   - affiliate_link: Buy link (where user is redirected)
   - is_featured: 1=Featured, 0=Normal
   - created_at: Creation date
   - updated_at: Last update

2. categories
   - id: Category ID
   - name: Category name (e.g., "Mobile Covers")
   - description: Category description
   - created_at: Creation date

3. admin_users
   - id: User ID
   - username: Login username
   - password: Login password
   - email: Email address
   - created_at: Creation date

4. contact_messages
   - id: Message ID
   - name: Sender name
   - email: Sender email
   - message: Message content
   - created_at: Submission date

API ENDPOINTS
=============

All API calls return JSON responses.

GET /includes/api.php?action=featured
- Returns 6 featured products

GET /includes/api.php?action=products
- Returns all products

GET /includes/api.php?action=products&category=Mobile+Covers
- Returns products from specific category

GET /includes/api.php?action=categories
- Returns all categories

GET /includes/api.php?action=product&id=1
- Returns single product details

POST /includes/api.php?action=contact
- Saves contact form message
- Body: {"name":"", "email":"", "message":""}

TROUBLESHOOTING
===============

ISSUE: Website shows blank page
FIX:
1. Check if Apache/MySQL are running
2. Check file permissions
3. Check browser console (F12) for errors
4. Check PHP error log

ISSUE: Database not connecting
FIX:
1. Start MySQL in XAMPP
2. Check credentials in config.php
3. Check if adrox_ecommerce database exists
4. Run setup.php again

ISSUE: 3D animations not showing
FIX:
1. Update browser (use Chrome, Firefox, Safari)
2. Check graphics card/GPU
3. Clear browser cache
4. Check console for WebGL errors

ISSUE: Products not loading
FIX:
1. Run setup.php
2. Check database connection
3. Verify products table exists
4. Check affiliate links are valid URLs

ISSUE: Admin login fails
FIX:
1. Username: admin
2. Password: admin123
3. Both must be exact match
4. Check for typos

SECURITY NOTES
==============

DEVELOPMENT (LOCAL):
- Admin password hardcoded
- SQL queries use basic escaping
- HTTP only (no HTTPS)
- Not meant for production

PRODUCTION CONSIDERATIONS:
1. Use password_hash() for admin passwords
2. Use prepared statements for all queries
3. Implement SSL/HTTPS
4. Add CSRF token protection
5. Add input validation on client & server
6. Use environment variables for secrets
7. Add rate limiting
8. Log all admin actions
9. Implement proper authentication
10. Regular security audits
